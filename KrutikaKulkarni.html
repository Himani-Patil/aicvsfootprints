Getting Started With Kaggle

By Krutika Kulkarni

Introduction

AICVS continued with its webinar series by hosting a session on the topic ‘Getting Started with Kaggle’ to brief the machine Learnings Newbies about datasets, data pre-processing and various machine learning algorithms including Linear Regression, Logistic Regression and Support Vector Machines. The session was delivered by a Machine Learning and deep Learning enthusiast Krutika Kulkarni, who also interned at Walmart Labs and was the former Head of Projects and Internships at AICVS.

What is Kaggle?

The Session kicked off with a brief introduction of Kaggle i.e., an online community of data scientists and machine learning enthusiasts. Kaggle allows users to find and publish data sets, explore and build models in a web-based data-science environment, work with other data scientists and machine learning engineers and enter competitions to solve data science challenges.

How to go about Working with Datasets

This was shortly followed by the discussion of steps to be kept in mind while working with datasets. The various steps are listed as follows:
1. Understanding and identifying the type of problem.
2. Loading the dataset and necessary modules.
3. Carrying out EDA to gain insights from the given data (this involves the preprocessing of closely observing the existing data, analyzing it and gaining insights from it).
4. Carrying out data pre-Processing.
5: Building and evaluating a model.

Building and Evaluating the Model

Furthermore, the various topics under building and evaluation a Machine Learning model specifically a Breast Cancer model which was demonstrated by Krutika were discussed which are summarized as followed:

1.Linear Regression: A supervised Learning algorithm that performs the task to predict a dependent variable value based on a given independent variable. A cost function is usually used to predict a particular output in this type of model. In the Breast-cancer model the Mean Squared error function is being used.

2.Logistic Regression: A supervised machine Learning algorithm used to determine the particular class to which a datapoint belongs. It is implemented when the target variable is categorical in nature. A sigmoid function is used to obtain the desired output in the Breast-cancer model.

3.Support Vector Machines: A type of supervised ML Algorithm which is mostly used for classification problems but can also be applied to regression models. Each data item plotted as a point in n-dimensional space with the value of each feature being the value of a particular coordinate. Thus, classification can be performed by finding the hyper-plane that differentiates the two classes very well. The hyperplane is chosen such that it maximizes the margin between the given two points. SVMs can be used for both linearly separable as well as non-linearly separable data points.

4. Random Forest: A supervised Learning algorithm that can be used for both regression and classification tasks. It is also a type of Ensemble Learning technique. Ensemble Learning is a machine learning Technique that combines many base models to create one robust optimal model which gives accurate results. It is an ensemble of many decision trees. All weak classifiers come together to make a strong classifier. While making predictions all the trees vote for the class they predict and then the class that secures the majority votes is taken as the final prediction.

5. Artificial Neural Networks: A neural network is a series of algorithms that endeavors to recognize underlying relationships in a set of data through a process that mimics the way human brain operates. Similar to neurons in the human brain, ANNs consist of perceptrons which form the basic units of a neural network. Any ANN consists of 4 components namely:

a.) Input Values
b.) Weights and bias
c.) Cost function
d.) Activation
The Above algorithms and Evaluation methods were discussed with regards to the Breast Cancer Prediction model and the Gap minder dataset both of which were discussed in a detailed and comprehensive manner.

Submissions in Kaggle

Lastly, Krutika talked about making submissions in Kaggle competitions by saving the machine learning model in a ‘y_pred’ variable and creating the submission csv file.

Conclusion


The session was wrapped up by an inspiring quote by the speaker which says “An Oops is a million times better than a what if” encouraging the beginners to experiment with new domains, make mistakes and take up new challenges.
